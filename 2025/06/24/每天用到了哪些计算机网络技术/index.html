每天用到了哪些计算机网络技术

发表于 2025-06-24 分类于 技术
本文字数： 1.2k 阅读时长 ≈ 1 分钟
网络知识的简单收集

你每天用的网络，其实都用到了哪些关键技术？

🧱 网络通信的基础概念（你肯定见过）

场景 背后知识点
打开浏览器访问网页（如百度） DNS解析、TCP连接、HTTP请求
使用微信或QQ聊天 长连接（TCP）、心跳包、NAT穿透
玩游戏（如 LOL） UDP通信、延迟、丢包、服务器同步
使用 API 或开发后端接口 RESTful API、HTTP状态码、CORS、安全头
  • 你的大多数服务，基本流程都是

🧰 常用协议和术语你应该掌握的（带图景）

协议/术语 实际作用 示例 建议理解深度
IP 唯一地址，定位设备 192.168.1.1，公网ip 了解ipv4/ipv6的概念
DNS 域名->ip解析 baidu.com-> 110.242.68.4 理解本地 DNS 缓存、污染问题
TCP/UDP 传输层协议 TCP 保证可靠、UDP 快 知道三次握手、UDP适用于游戏
HTTP/HTTPS 访问网页用的协议 GET/POST 请求，状态码 状态码、请求头、SSL 证书
port 服务“编号” 80 是 HTTP，443 是 HTTPS 前后端服务调试时常用
NAT 路由器内网映射 家用网络常见，端口映射 搞内网穿透时必须了解

举个例子：你访问 https://steamcommunity.com/ 实际经历了什么？

  1. 浏览器先问 DNS：steamcommunity.com 是哪个 IP？
  2. 得到 IP 后，发起 TCP 三次握手，建立连接。
  3. 客户端发起 TLS 握手，进入 HTTPS 通信。
  4. 发出 GET 请求：GET /market/。
  5. 服务器响应 HTML/CSS/JS，浏览器渲染页面。

🕸️ 前后端常用网络知识

内容 实际作用 实例
HTTP 状态码 判断请求是否成功 200 成功、404 找不到、500 服务错误
CORS 跨域 限制不同域之间访问资源 本地调接口时报错：CORS policy blocked…
Cookie / Session / Token 用户身份验证 登录之后保持会话
HTTP Method 区分动作类型 GET（查），POST（增），PUT（改），DELETE（删）

🚀 网络性能优化相关知识（加速器相关）

技术点 实际作用
CDN 加速 靠近你的位置提供缓存，提高访问速度
DNS 优化 减少解析时间，防污染
反向代理 缓存热点资源，减少后端压力（如 nginx）
连接复用（HTTP/2） 一个连接跑多个请求，提升效率
TLS 握手优化 降低首次 HTTPS 连接的延迟
  • Steam++ 类工具的原理其实就用到了这些技术：
  • DNS优化 + 智能选路 + 连接复用 + 本地代理分流。

🔐 网络安全也与你相关（别等被攻击才学习）

概念 示例 你能做什么
XSS / CSRF 你的网站被嵌入恶意脚本 后端加 Token 验证，前端设置 CSP
数据包监听 不加密传输被人窃取 用 HTTPS
拒绝服务（DoS） 短时间内请求爆炸 限流、熔断、缓存
